# Case 验收记录 001 — empyrical #60 (alpha 年化)

## 结论:淘汰(环境成本过高)

> 专业性与 Oracle 质量高,但项目年代久(2019),环境构建需处理多个与目标 Bug 无关的历史兼容问题,不符合"低成本、可规模化"的样本生产要求。

## Case 概况

| 项 | 内容 |
|---|---|
| 仓库 | quantopian/empyrical |
| base commit(修复前) | 92011b7 |
| fix commit(修复后) | 4cb0508 `BUG: fix annualization of alpha (#60)` |
| Bug 本质 | alpha 年化:错误用**线性**年化 `mean(alpha)×252`,正确应为**复利** `(1+mean(alpha))^252−1` |
| 数值差异 | 错误 −8.31 vs 正确 −0.9998(方向性明显,−831% alpha 是荒谬值) |
| 源码+测试 | ✅ PR 同时改了 stats.py 与 test_stats.py,oracle 现成 |

## 验收维度打分

| 维度 | 结论 |
|---|---|
| 专业性(金融语义) | 高 —— 线性 vs 复利年化,真金融方法理解 |
| GT 可靠性 | 高 —— 维护者合并的修复 + 回归测试 |
| Oracle 清晰度 | 高 —— 测试期望值明确 |
| 离线性 | 高 —— 纯 numpy,小固定序列,不需网络 |
| **环境健康度** | **低 —— 致命伤** |
| 非目标兼容债 | 多 |
| 最终判定 | **淘汰** |

## 环境债务清单(踩坑实录)

1. 本地 pandas 2.3.3 禁止 `Series[:, np.newaxis]`,旧代码 `beta_aligned` 直接报错 —— 与 alpha bug 无关
2. 尝试装同期 pandas 0.25.3 → 无预编译 wheel,需源码编译
3. 源码编译缺 `pkg_resources`(setuptools)→ 连锁失败
4. 即便装好中间版 pandas,仍可能继续遇到其他 API 弃用错误

以上全部属于**样本环境债**,不是"Agent 要解决的 alpha 年化 bug"。若正式交给 Agent,这些环境错误会掩盖真正的能力信号——无法区分"Agent 不会修 alpha"还是"被 pandas 兼容卡住"。

## 由本 Case 反推出的【环境筛选硬条件】

未来候选 Case 必须同时满足,否则淘汰:

1. **当前主流 Python 版本(3.10+)可直接安装**,不需为其配历史解释器
2. **不需源码编译多个历史依赖**(依赖有预编译 wheel)
3. **原始测试框架可直接运行**
4. **不需修改与目标 Bug 无关的源码**即可跑通
5. **优先近三年仍活跃维护**、带 `pyproject.toml` / lockfile / CI 配置的仓库

> 核心规则:**环境健康度是硬门槛,不是附加项。** 一个 bug 金融语义再漂亮,若跑起来要抢救一个老项目,它的"可规模化"就是假的。
