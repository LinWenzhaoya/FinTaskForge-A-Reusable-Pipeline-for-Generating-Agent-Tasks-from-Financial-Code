# 局限性与诚实边界

本文件如实记录本次交付的边界，避免过度声称。

## 1. 隔离不是 OS 级

正式 run 03 的隔离由三部分构成：
- **独立临时 workspace**（`/tmp` 下，只含 `task/*`）；
- **空 `CLAUDE_CONFIG_DIR`**（不加载任何全局 CLAUDE.md / 历史偏好）；
- **上下文隔离**（全新会话、白名单环境变量、cwd 限定在 workspace、不 `--add-dir` 其他目录）。

它**不是**：
- 容器/OS 级文件系统隔离；
- 强制断网。

因此**不能声称"Agent 技术上无法访问其他宿主路径"**。`--allowedTools` 是对指定工具的**自动授权**，并显式禁用了 WebSearch/WebFetch/Task；但 CLI 初始化的运行时工具列表仍含其他工具，且 Bash 理论上仍可执行 `curl` 等联网命令——**禁止联网是 Prompt 与工具层约束，不是技术断网**。实际运行中唯一的联网表现是离线环境下数据集下载超时（与修复无关）。

## 2. completeness Oracle 存在边界解释空间

`semi_variance` 是否属于**必要**修复范围可讨论：
- **支持纳入**：官方 fix 同步改了 variance 与 semi_variance；两处缺陷模式完全相同；TASK 要求"模块内同类度量一致性"，Prompt 要求"检查相同或高度相关逻辑"。
- **Agent 的反驳**：用户仅报告 variance；semi_variance 是另一公开 API、公开测试未覆盖、其默认阈值语义可视为独立行为。

故该项定义为 **completeness（关联行为完整性）测试**，而非无争议的核心正确性，也非 anti-hack。核心正确性 Oracle（variance target + regression）是强的、无争议的。

## 3. 正式运行耗时较长

run 03 约 1h55m、67 轮，主因是 Agent **主动**跑了大量 skfolio 回归模块（cvxpy 凸求解慢）。这暴露了正式 Harness 未来应具备：总墙钟限制、单条测试命令超时、推荐回归范围、超时后可审计的终止状态。本次不重跑，保留真实结果。

## 4. 规模化尚未完成

- 候选缺陷发现仍以人工为主（领域价值判断不可自动）；
- 隐藏测试设计依赖领域专家；
- 仅验证**同仓库**复用（001/002 均来自 skfolio），未验证跨仓库/跨语言/跨材料类型；
- 未做批量质量门禁、难度分层校准、多 Agent 通过率标定、规模化成本指标统计。

成熟度：**L2（可复用流程）完成，L3（半自动管线）初步具备，L4（规模化生产）未完成**。定位为"可复用造题管线原型"，非"成熟规模化平台"。

## 5. 正式实验前的两次无效运行（审计附注）

正式 run 03 之前进行了两次基础设施校验，均被判**无效运行**、不计入正式 Agent 测试：
- **run 01**：启动脚本未将 Agent 的工作目录切到隔离 workspace，导致 cwd 落在 benchmark 内部、隔离不成立；
- **run 02**：编排层的 cwd 门禁对 `/tmp` 与 `/private/tmp` 符号链接做了朴素字符串比较，误判并中止，模型 0 工具调用、未开始解题。

二者均属**编排错误**而非模型解题失败；原始证据保留在项目内部审计目录，未纳入本提交包。预注册纪律：一旦模型开始有效输出/调用工具即视为唯一正式运行，不重跑择优——run 03 即该唯一有效运行。
