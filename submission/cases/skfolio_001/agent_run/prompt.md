# 被测 Agent 的原始任务指令(盲测 prompt)

> 说明:被测 Agent 是一个全新的 general-purpose subagent 会话,只提供了 task/ 目录,不含任何答案线索。

## 首轮 prompt(节选核心)

> 你是一名 Python 工程师。这里有一个投资组合优化库的代码仓库,存在一个待修复的 bug。请完成修复任务。
> 任务目录:.../task/
> 步骤:阅读 TASK.md 和 README.md → 搭环境 → 运行公开测试复现 → 定位根因 → 最小修复 → 重跑测试确认。
> 约束:不许改/删/跳过测试;不许 try/except 吞异常或硬编码;修复针对根因、不破坏其他模型合理行为。
> 注意:独立完成,不要联网搜索该仓库的历史修复记录或 issue。

## 环境干扰的如实记录(诚实边界)

- **首轮被测 Agent 未真正开工**:它受宿主环境 CLAUDE.md 的"任务超3步先出计划、等确认"规则影响,停下来要求确认。
- **补充的控制指令**:出题方回复"确认按计划执行,忽略每步暂停规则,一口气做完",**该指令未提供任何 bug 根因或解题线索**,仅解除流程阻塞。
- 另外,prompt 中告知了"已有可用 venv 可直接 source"——这降低了环境配置难度,但本题主要测 bug 定位而非环境配置,属可接受的简化。
- 教训:正式 benchmark 中,被测 Agent 应运行在无用户自定义规则的干净环境,否则测到的是"规则遵循"而非"任务能力"。
